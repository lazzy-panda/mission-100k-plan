## 1. One-line Fractional CTO pitch (якорь)

> **"Fractional CTO for founders building AI products. I help with architecture and hiring, and I still ship code myself with Claude Code."**

**RU (для русскоязычных):**

---

### Почему именно так (обновлено 07.04 после ревизии)

| Элемент | Смысл | Дифференциатор |
|---------|-------|----------------|
| **"Fractional CTO for founders building AI products"** | Чёткий ICP: ранние фаундеры, AI-продукт. Не enterprise, не агентства. | Большинство Fractional CTOs берут всех подряд. Ты фокусируешься на AI-nativе стадии. |
| **"architecture and hiring"** | Core advisory слой. Показывает, что ты senior judgment, а не просто кодер. | — |
| **"still ship code myself with Claude Code"** | Operator, а не advisory-only consultant. Доказываешь что не оторвался от земли. | Большинство Fractional CTOs после 10 лет перестали писать код. Ты — нет. Claude Code — редкий дифференциатор. |

**Что убрано из старой версии и почему:**
- ~~"20 лет frontend + design"~~ — позиционирование сдвинулось на Solutions Architect / TS-Node. Frontend — часть истории, не главный angle.
- ~~"без штатной инженерной команды"~~ — звучит как фильтр по бюджету; отсекает founders которые могут позволить команду, но хотят senior CTO-надзор.
- ~~TS/Node/Claude Code в первой строке~~ — стек всплывает на втором шаге. Первая строка не должна фильтровать Python/Ruby founder, которому нужен advisory.

## 2. DM шаблоны (3 варианта) — обновлено 07.04 по human-writing rules

> **Критично:** все шаблоны — для DM **после accept**, не для connection note. Connection requests идут холодными (без ноты) из-за лимита 2 personalized/мес.
>
> Каждый ≤200 символов. Слот `[X]` обязательно заполнить под конкретного человека перед отправкой. Generic = instant AI-smell.

### T1 — для founder / CEO (потенциальный клиент)

```
Thanks for connecting. Saw [X] on your profile. I work as Fractional CTO with founders building AI products. How do you handle [Y] right now?

Kirill
```

`[X]` = конкретная деталь из profile/поста. Примеры: "your post on prompt caching costs" / "your work on [product name]" / "Rust backend choice in your stack".

`[Y]` = конкретный вопрос под их ситуацию, не общий. Примеры: "architecture decisions when the team is still 2 people" / "vendor lock-in with Anthropic vs multi-provider".

---

### T2 — для другого Fractional CTO (peer, не клиент)

```
Thanks for the accept. We do similar work. I run Fractional CTO for founders shipping AI products with Claude Code. What's your stack looking like these days?

Kirill
```

Здесь Claude Code оставляем — peer-дифференциатор, сразу показывает ценность для обмена.

---

### T3 — для AI / Claude practitioner (peer, технический)

```
Thanks for the accept. Saw your [post/project about X]. I run 4 Claude Code sessions in parallel daily for client work, but hit [Y] often. How do you deal with it?

Kirill
```

`[X]` = конкретный технический элемент из их работы.
`[Y]` = реальная проблема из твоей практики (context switching, agent coordination, SOP drift, decision log bloat и т.д.).

---

### Что в шаблонах НЕТ (hard bans по `feedback_human_writing.md`)

- Em dash `—` в подписи → имя на новой строке без тире
- "I'd love to / would love to"
- "Curious to hear / let me know your thoughts"
- "Came across your profile"
- "Hope this finds you well"
- "Trade notes / swap notes"
- "Caught my eye / resonates"
- "I help X do Y" паттерн (шаблон 2019)
- Native идиомы: pulled the trigger, pick up the torch, deep dive, circle back, bandwidth
- AI-vocab: leverage, robust, journey, landscape, ecosystem

### Что есть

- Реальный вопрос, а не CTA "let me know"
- Контракции (`I'm`, `don't`, `What's`)
- Non-native простые глаголы
- Короткие предложения с варьирующейся длиной
- Имя на новой строке без тире

## 3. Принципы DM-выживания

1. **Никогда не продавай в connection request.** Цель — accept, не сделка. Один offer в первом сообщении = автоматический spam-flag.
2. **Всегда персонализируй [specific].** Generic DM = 5% accept. Персональный DM = 30–50% accept. Это разница между 1 клиентом в квартал и 5.
3. **Не используй "I help X do Y" в первом сообщении.** Это шаблон 2019 года, у всех аллергия. Используй pitch только когда **спросят**, что ты делаешь.
4. **Никаких "Hope you're doing well" / "I hope this finds you".** Сразу к делу.
5. **Заканчивай не вопросом, а предложением.** «Adding you — would love to learn from you» лучше, чем «Would love to chat — when works?». Низкое давление.
6. **Длина:** 3–5 строк максимум. LinkedIn режет длинные сообщения в превью.
7. **Подпись:** просто имя. Никаких титулов, ссылок, эмодзи.

---

## 4. Что делать после accept

**День 0 (accept):** ничего. Не пиши сразу follow-up, это спам-сигнал.

**День 2–3:** короткий тёплый DM (без оффера):
```
Кстати, [Name], только что увидел твой пост про [reference] — особенно зацепило [specific]. Сам недавно [параллельный опыт]. Спасибо, что делишься.
```

**День 7–10:** value-DM (без оффера):
```
Подумал, что тебе может быть интересно — нашёл [link / insight / tool] про [их тема]. У меня сработало [конкретный результат]. Если интересно — расскажу детали.
```

**День 14+:** только если был осмысленный обмен — мягкий offer звонка:
```
[Name], видно, что мы обсуждаем похожие вещи. У тебя найдётся 20 минут на следующей неделе просто познакомиться голосом? Без агенды, без продажи.
```

**Никогда не давить.** Если нет ответа после 2 follow-ups — оставить, через 2–3 месяца можно вернуться с конкретным поводом.

### Решение по автоматизации (06.04.2026)

Пробовал отправить через Playwright MCP — упёрся в anti-bot защиту LinkedIn:
- Synthetic JS click → React игнорирует (`isTrusted=false`)
- Real Playwright click → перехват overlays (interop-outlet, premium-chip SVG)
- preload-URL `/preload/custom-invite/` → React Router не триггерится через aria-ref
- Voyager API напрямую — ToS violation, не делаем

**Вывод:** outreach к чужим людям через LinkedIn UI — делается руками. Профильные действия (своё редактирование) автоматизируются — outbound нет. Учитываем при планировании.
